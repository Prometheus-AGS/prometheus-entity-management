# Specification: @prometheus-ags/prometheus-entity-management v1.0.0

**Status:** Locked for release `1.0.0`  
**Scope:** Public npm package, production consumption, AI-agent skills alignment.

## 1. Purpose

Deliver a **normalized, globally reactive entity graph** for React (Zustand + Immer) so one canonical record per `(entityType, id)` updates every list, detail, and badge that references it. **v1.0** is **Zustand-only** for the store layer (no Redux/MobX backends).

## 2. Functional completeness (must ship in 1.0.0)

### 2.1 Library surface

- **Graph:** `entities`, `patches`, `lists`, `entityStates`; `upsertEntity`, `readEntity`, list mutations, patches.
- **Engine:** `configureEngine`, deduplicated fetches (`dedupe`), optional garbage collection, `fetchEntity` / `fetchList` semantics.
- **Hooks:** `useEntity`, `useEntityList`, `useEntityMutation`, `useEntityAugment`, `useSuspenseEntity`, `useSuspenseEntityList`.
- **View / CRUD / GraphQL / adapters:** As exported from [`src/index.ts`](../../src/index.ts); realtime adapters and `RealtimeManager` coalescing.
- **DevTools:** `useGraphDevTools` exported for debugging (optional UI in consumers).

### 2.2 Quality bar

- **Typecheck:** `pnpm run typecheck` passes on the library.
- **Build:** `pnpm run build` produces `dist/` with ESM + CJS + types.
- **Tests:** Automated smoke tests (Vitest) for graph, engine `dedupe`, and at least one realtime path.
- **CI:** GitHub Actions runs install, library typecheck, build, test, `verify:skills`, and typecheck for Vite + Next.js examples on PR/push to `main`.
- **Skills sync:** `pnpm run verify:skills` compares runtime export names from `dist/index.mjs` to the committed ledger `skills/_shared/references/library-exports.json` (refresh after API changes with `pnpm run refresh:exports`).

### 2.3 Documentation

- **README:** Accurate feature comparison, bundle size methodology, links to guides.
- **CLAUDE.md / AGENTS.md:** No contradictions with implementation (Suspense, GC, DevTools, tests).
- **Guides:** TanStack Query + React Table coexistence; advanced recipes (ElectricSQL beta, WebRTC as `ChangeSet` adapter pattern).
- **RELEASING.md + CHANGELOG:** Semver and publish checklist.

### 2.4 Examples

- **Vite + Next.js** examples build and document run commands; Vite includes a **TanStack Query → graph** bridge demo route.

### 2.5 Distribution

- **npm:** Package `@prometheus-ags/prometheus-entity-management` at version **1.0.0** with `prepublishOnly` running typecheck + build + test + `verify:skills`.

## 3. Non-goals for v1.0

- First-class Redux, MobX, or non-Zustand graph backends.
- Built-in WebRTC CRDT sync (document adapter pattern only).
- Subpath package exports (optional follow-up).
